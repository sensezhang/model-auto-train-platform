# 实时训练指标可视化功能

## ✅ 功能说明

现在你可以在训练过程中实时查看损失曲线和mAP指标了！不需要等到训练完成。

### 新增功能

#### 1. **实时损失曲线 (Loss Chart)**
- 显示每个epoch的训练损失
- 红色曲线，实时更新
- 帮助你判断模型是否正在收敛

#### 2. **mAP精度曲线 (mAP Charts)**
- **mAP50**: 蓝色曲线 (IoU=0.5的平均精度)
- **mAP50-95**: 绿色曲线 (IoU从0.5到0.95的平均精度)
- **mAP**: 紫色曲线 (RF-DETR的mAP指标)

#### 3. **自动更新**
- 每5秒自动刷新一次指标
- 显示最近100个epoch的数据
- 无需手动刷新页面

## 🚀 使用方法

### 第一步：启动训练

1. 进入**训练中心**
2. 创建并启动一个训练任务

### 第二步：查看实时指标

1. 点击选择正在运行的训练任务
2. 在右侧任务详情中，会看到**两个图表区域**：
   - 📉 **损失曲线** - 顶部，红色
   - 📈 **mAP曲线** - 底部，蓝/绿/紫色

### 第三步：监控训练

训练过程中，图表会**自动更新**：
- 每完成一个epoch，新的数据点会自动添加到图表中
- 曲线会实时延展
- 你可以看到训练是否正常收敛

## 📊 图表说明

### Loss曲线解读

```
高 ┃╲
   ┃ ╲
   ┃  ╲___
   ┃      ╲___
低 ┃          ╲___________  ← 理想的loss曲线（持续下降）
   └─────────────────────→ Epoch
```

**正常表现：**
- ✅ Loss持续下降
- ✅ 曲线逐渐平缓
- ✅ 最终趋于稳定

**异常表现：**
- ⚠️ Loss不降反升 → 学习率可能过高
- ⚠️ Loss剧烈震荡 → Batch size可能过小
- ⚠️ Loss一直很高 → 数据或模型可能有问题

### mAP曲线解读

```
1.0┃             ________  ← 理想的mAP曲线（持续上升）
   ┃          __/
   ┃       __/
   ┃    __/
0.0┃___/
   └─────────────────────→ Epoch
```

**正常表现：**
- ✅ mAP持续上升
- ✅ 逐渐接近1.0
- ✅ 最终趋于稳定

**目标值：**
- 🎯 mAP50 > 0.8 (80%) - 优秀
- 🎯 mAP50-95 > 0.5 (50%) - 良好

## 🎯 使用场景

### 1. 训练监控
**问题：** 不确定训练是否正常进行？
**解决：** 实时查看loss曲线，如果持续下降说明正常

### 2. 早停判断
**问题：** 不知道什么时候停止训练？
**解决：** 当mAP曲线不再上升时，可以考虑提前停止

### 3. 参数调优
**问题：** 不知道超参数设置是否合适？
**解决：** 通过观察曲线的稳定性和收敛速度判断

### 4. 问题诊断
**问题：** 训练效果不好，不知道原因？
**解决：**
- Loss不降 → 检查学习率、数据质量
- mAP不升 → 检查标注准确性、类别平衡

## 💡 最佳实践

### 训练前
1. 先用少量epoch（如10个）测试
2. 观察loss是否开始下降
3. 如果正常，再开始长时间训练

### 训练中
1. 定期查看图表（每小时）
2. 如果loss突然上升，立即停止检查
3. 如果mAP长时间不变，考虑调整学习率

### 训练后
1. 保存最终的指标图表（截图）
2. 对比不同模型的训练曲线
3. 选择表现最好的模型

## 🐛 故障排除

### 问题1：看不到图表

**症状：** 选择任务后没有显示图表

**原因：**
- 训练刚开始，还没有epoch数据
- 日志文件解析失败

**解决：**
1. 等待第一个epoch完成
2. 检查训练日志是否正常
3. 刷新页面重试

### 问题2：图表数据不更新

**症状：** 图表显示但不更新

**原因：**
- 自动刷新被禁用
- 训练已完成或失败

**解决：**
1. 确认训练状态为"running"
2. 检查网络连接
3. 打开浏览器控制台查看API错误

### 问题3：只有Loss没有mAP

**症状：** 只显示损失曲线，没有mAP曲线

**原因：**
- 前几个epoch可能还未计算mAP
- YOLO和RF-DETR的指标名称不同

**解决：**
- 正常现象，等待更多epoch完成
- mAP通常在验证阶段才计算
